# 当前产品实现审计

更新时间：2026-07-18

本文件是当前代码“哪些可以保留、哪些需要升级、哪些尚未开发”的唯一判断入口。产品目标以 `PRODUCT_BASELINE.md` 为准，开发路线以 `ROADMAP.md` 为准；旧聊天结论和历史 M0～M8 编号不再单独决定进度。

## 判断规则

- **正式地基**：结构符合产品与架构方向，后续可以增量扩展，不需要推倒重来。
- **首条可用闭环**：真实功能已经能运行，但覆盖面、质量或用户流程仍未达到阶段验收。
- **需升级**：可以保留部分代码，但当前结构不能直接作为正式产品继续扩展。
- **尚未开发**：只有名称、设计或阶段位置，没有用户可用功能。

## 后续开发准入规则

每个用户功能开始编码前必须完成以下检查：

1. 在 `PRODUCT_BASELINE.md` 中确认用户目标和验收位置。
2. 在 `RESEARCH_TRACEABILITY.md` 中确认对应参考项目、采用机制和明确拒绝的做法。
3. 先写清页面入口、用户操作、成功状态、失败状态和恢复方式。
4. 分阶段实现可以缩小功能范围，但数据结构和页面位置必须符合正式产品方向。
5. 模拟数据、临时表单和内部编号只能出现在调试入口；未开发功能必须明确标注，不能冒充已完成。
6. 自动测试通过后，只有达到用户可执行的完整流程才进入用户验收。
