# 产品基线 V0.2

## 产品目标

本项目要交付的是一个可长期浏览和操作的中文小说拆解工作台，不是一次性报告生成器。

用户一次导入整本小说，系统根据篇幅自动分层处理。首次拆解按阶段引导，用户查看阶段总览、抽查原文并确认后进入下一步。完成后，用户可以在人物、剧情、事件、时间线、伏笔、冲突、节奏和世界设定等版块之间浏览、搜索和跳转。

每项重要分析必须能够回到来源章节和原文上下文。报告与文件导出是附加能力，不是产品主体。

## 首版边界

- 中文小说优先，单用户、本机浏览器使用。
- 支持短篇、中长篇和百万字以上长篇，不设置统一小说字数上限。
- 第一批导入格式：TXT、Markdown、DOCX、EPUB。
- 小说和项目数据保存在本机；首版使用用户配置的在线 LLM 进行分析。
- 质量优先，长篇分析允许持续数小时，并且必须显示阶段、进度、失败原因和下一步。
- 第一版先完成核心内容；主题、文风、写作技巧和正式报告导出后续增加。

## 用户流程

1. 创建小说项目并导入整本小说。
2. 检查并确认卷章结构、顺序、空章、重复章和异常内容。
3. 检查并确认人物识别结果。
4. 检查并确认剧情与事件结果。
5. 检查并确认事实、世界设定和人物状态变化。
6. 检查并确认伏笔、冲突、节奏和场景功能。
7. 进入完整工作台，自由浏览各版块并随时回到原文。

每个阶段默认停在确认页。用户可以确认进入下一步，也可以标记具体问题。开发阶段优先定位问题来自原文解析、系统规则、Prompt 还是模型能力，不把人工修正当作掩盖系统问题的默认方案。

## 统一阶段

仓库只使用下面这一套产品阶段。研究文档中的同名旧阶段仅作为历史记录，不再表示当前开发进度。

| 阶段 | 用户可见目标 | 当前状态 |
|---|---|---|
| P0 工程地基 | 后台任务、恢复、制品和自动测试可靠 | 基本完成 |
| P1 整本导入 | 四种文件导入、卷章识别、原文证据和章节确认 | 未开始 |
| P2 人物与事件 | 真实 AI 识别人物、剧情与事件，并可回到原文 | 未开始 |
| P3 事实与状态 | 世界设定、事实变化、人物状态和时间查询 | 未开始 |
| P4 核心分析 | 伏笔、冲突、节奏和场景功能 | 未开始 |
| P5 拆解工作台 | 各版块浏览、搜索、关联跳转、阶段确认 | 未开始 |
| P6 长篇验收 | 短篇、中篇、长篇、百万字样本端到端验收 | 未开始 |

## 用户验收边界

后台任务编号、租约代次、数据库迁移、分支、PR 和自动化测试不需要最终用户验收。

只在下面三个节点邀请最终用户试用：

1. 能上传真实小说、正确显示章节，并得到真实人物和事件结果。
2. 核心分析版块可以完整浏览，并能从结果跳回原文。
3. 整本长篇能够从导入连续运行到完整工作台。

## 当前真实进度

研究和候选架构已经形成，P0 工程地基的关键可靠性测试已经通过。当前仓库仍只有内部任务调试台，没有整本小说导入和真实拆解功能。按最终产品能力估算，当前约完成 10% 到 15%。
